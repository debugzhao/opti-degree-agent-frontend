# 志愿规划 Agent 项目摘要

## 项目定位

本项目是面向高考生和家长的「AI 志愿决策助理 + 专家复核咨询平台」原型。产品目标不是做普通聊天机器人，而是做可解释、可追溯、可复核的志愿填报决策系统：先通过免费测算建立信任，再引导用户补全家庭背景和偏好，生成冲稳保方案、志愿表风险体检，并在高风险或高客单价场景中进入顾问复核。

核心原则：

- 不承诺录取，不使用「保录」「必中」「精准录取」「内部数据」等话术。
- AI 只做辅助决策，最终填报必须回到省级考试院官方系统。
- 关键推荐必须绑定数据源、年份、省份、位次、专业组、规则命中等证据链。
- 高风险结论必须触发人工复核。
- 免费工具解决「查什么」，本产品解决「怎么选、风险在哪里、家庭如何达成一致」。

## 当前实现状态

当前仓库是 Phase 0 React 单页原型，使用模拟数据和本地推荐逻辑完成基础业务闭环演示。README 中的 PRD 规划了未来的 Taro 多端前端、FastAPI 后端、LangGraph 多 Agent、RAG、向量数据库、订单支付、顾问工作台等能力，但这些后端和 Agent 服务尚未在当前代码中实现。

已实现的前端流程：

- 测算：输入省份、性别、批次、分数、位次、选科，生成免费风险画像。
- 建档：填写家庭类型、预算、决策优先级、风险风格、专业/城市偏好、不可接受专业、是否接受外省和读研。
- 方案：生成保守型、均衡型、进取型三套冲稳保方案，展示录取概率、综合分、趋势分、学费、推荐理由。
- 付费解锁模拟：选择 AI 初版报告、深度报告、顾问复核、填报陪跑套餐后展示证据链和更多操作。
- 志愿表风险体检：对当前方案检查保底不足、热门专业集中、不可接受专业、梯度过密。
- 顾问工作台：高价套餐下展示 AI 咨询底稿、复核清单和家庭会议议程。
- 家庭协同：学生、父亲、母亲分别标注喜欢、不能接受、有疑问，并提示分歧。

## 技术栈和运行方式

- 前端：React 19 + Vite 7。
- 图标：lucide-react。
- 当前入口：`src/main.jsx` 挂载 `src/App.jsx`。
- 样式：纯 CSS，集中在 `src/styles.css`。
- 数据与算法均在前端本地模拟，没有真实 API 调用。

常用命令：

```bash
npm run dev
npm run build
npm run preview
```

`npm run dev` 会以 `127.0.0.1` 启动 Vite，本地默认地址通常是 `http://127.0.0.1:5173/`。

## 代码结构

- `README.md`：业务导向 + 技术导向 PRD，是当前主要产品需求来源。`docs/` 目录目前没有实际 PRD 文件。
- `src/App.jsx`：单页应用主体，包含测算、建档、报告、顾问、家庭协同五个页面组件和页面状态。
- `src/recommender.js`：本地推荐和风险计算逻辑，包括初始表单数据、位次估算、风险画像、推荐方案、志愿表风险、顾问底稿。
- `src/data.js`：模拟配置和样本院校数据，包括省份、选科、性别专业适配、家庭类型、偏好选项、付费套餐、院校专业组。
- `src/styles.css`：页面布局、响应式、表单、卡片、按钮、标签、风险提示等样式。
- `src/main.jsx`：React 根节点渲染。
- `vite.config.js`：Vite React 插件配置。

## 当前推荐逻辑摘要

`src/recommender.js` 是原型核心。

- 位次：优先使用用户输入的省排名；没有位次时通过分数估算，默认约束在 4500 到 180000 之间。
- 风险画像：根据位次是否缺失、省份是否在首期深度覆盖、是否高分段、家庭背景是否补全、是否存在不可接受专业、选科组合等生成风险分和风险项。
- 支持深度覆盖省份：河南、山东、广东、江苏、浙江、四川。
- 推荐评分总分使用 PRD 中的权重思想：录取概率、专业适配、就业/趋势、城市偏好、成本风险综合计算。
- 录取概率是模拟分层，不是真实预测。真实上线必须接入官方招生计划、历年投档线、一分一段表、专业选科要求和招生章程。
- 方案分层：`保`、`稳`、`冲`、`高冲`，再组合成保守型、均衡型、进取型三套方案。
- 志愿表风险：检查保底不足、热门专业集中、不可接受专业命中、梯度过密；无明显问题时返回结构基本合理。
- 顾问底稿：基于均衡型方案输出考生概况、初步结论、复核清单和 30 分钟家庭会议议程。

## PRD 规划的未来架构

前端规划为 React + Taro，多端支持 H5 和微信小程序。后端规划为 FastAPI，后续接用户档案、报告、订单、顾问复核、家庭标注、证据来源等 API。

Agent 架构规划：

- Supervisor Agent：识别阶段、路由任务、聚合结论。
- Profile Agent：追问并补全学生与家庭档案。
- Retrieval Agent：检索招生、政策、专业、就业、行业证据。
- Policy Rule Agent：校验省份规则、选科、体检、单科、学费、批次。
- Recommendation Agent：生成冲稳保方案和排序。
- Risk Agent：检查滑档、退档、调剂、热门扎堆、保底不足。
- Career Trend Agent：分析专业与 5-10 年行业趋势。
- Report Agent：生成家长可读报告和证据链。
- Reflection Agent：检查过度承诺、数据缺失、风险漏报。
- Human Review Agent：生成顾问复核底稿和人工复核清单。

推荐范式是 Supervisor Multi-Agent + Plan and Solve 主流程 + ReAct 子 Agent + Reflection 质检 + Human-in-the-loop 复核。

## 数据与 RAG 规划

结构化强约束数据必须进入 PostgreSQL，不应只放向量库。RAG 用于解释、补充和非结构化证据，不替代硬规则。

PRD 规划的知识库包括：

- 招生计划库、历年投档线库、一分一段库、院校专业组库、选科要求库。
- 体检限制、招生章程、专业介绍、就业质量报告、行业趋势、顾问案例库。

MVP 向量数据库推荐 PostgreSQL + pgvector；中期可升级 Qdrant 或 Milvus。

## 商业路径

MVP 重点验证「免费风险画像 -> AI 报告 -> 顾问复核」漏斗，以及「志愿表风险体检」是否是强转化入口。

套餐设计：

- 免费：风险画像、数据完整度、省份覆盖。
- AI 初版报告：99 元，完整三套方案、基础证据链、基础风险体检。
- 深度报告：399 元，详细证据链、志愿表体检、AI 答疑。
- 顾问复核：999 元，人审结论、家庭会议、服务留痕。
- 填报陪跑：1999 元，两次复核、最终表检查、应急答疑。

## 合规与风控注意事项

开发和文案必须持续遵守：

- 禁止保证录取、必中、精准录取、内部数据、包过、保上。
- 禁止代替考试院填报。
- 禁止要求或保管用户官方系统密码。
- 禁止夸大专业就业收入或承诺未来薪资。
- 未成年人数据最小化采集，敏感信息加密存储。
- 上传图片、语音、PDF 需要过期清理策略。
- 顾问只能访问自己服务订单相关资料。
- Agent 输出必须经过合规检查，高风险报告必须进入 human-in-the-loop。
- 关键报告需保存 prompt、工具调用、证据来源、模型版本和生成时间。

## 后续开发提醒

- 当前 `src/data.js` 的院校、位次、证据链均为演示数据，不能当作真实招生建议。
- 接入真实后端前，前端应清晰区分「模拟演示」和「真实报告」状态。
- 任何正式推荐都应优先走结构化规则和官方数据，LLM 只能解释和补充。
- 新增页面时保持当前产品流程：测算 -> 建档 -> 方案 -> 体检/付费 -> 顾问/家庭协同。
- 处理高风险场景时，界面应提示人工复核，而不是直接给确定性结论。
